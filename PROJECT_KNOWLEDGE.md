## 🏗️ Architecture & Structure

### 1. Directory Structure
The repository is a monorepo-like structure but focused on one main app:
- `/portfolio-nextjs` → **Main Application Source Code** (Next.js).
  - `/src` → All components, pages, and styles.
  - `/public` → Static assets (images, fonts).
  - `Dockerfile` → Multi-stage build configuration.
- `/nginx` → Nginx configuration for VPS (`app.conf`).
- `/certbot` → SSL Certificates data (auto-generated).
- `.github/workflows` → CI/CD Pipelines.
- `docker-compose.yml` → Orchestration for Nginx + Certbot.

### 2. Deployment Workflow (CI/CD)
There are two active workflows in GitHub Actions:
1. **Deploy to VPS (`deploy-vps.yml`)**:
   - Trigger: Push to `master`.
   - Action: SSH into VPS (`43.157.xxx.xxx`), pull latest code, rebuild Docker container (`docker compose up -d --build`).
   - Secrets Required: `VPS_HOST`, `VPS_USER`, `VPS_SSH_KEY`.
   - **Critical:** VPS project path is `/home/ubuntu/rizqibennington`.

2. **Deploy to GitHub Pages (`deploy.yml`)**:
   - Trigger: Push to `master`.
   - Action: Build Next.js (`npm run build`), export to static (`out`), deploy to `gh-pages` branch.

### 3. Server Configuration (VPS)
- **OS:** Ubuntu
- **Reverse Proxy:** Nginx (running in Docker container `web-rizqi`).
- **SSL:** Let's Encrypt via Certbot (Auto-renew enabled in `docker-compose.yml`).

## 🛠️ Development Guidelines

### 1. Adding/Editing Content
- **Components:** All sections are in `portfolio-nextjs/src/components/sections/`.
- **Images:** Place in `portfolio-nextjs/public/images/`.
- **Styling:** Use Tailwind CSS classes. Global styles in `globals.css`.
- **Icons:** Use `lucide-react`.
- **Theme:** Uses `next-themes` for Dark/Light mode. Styling supports `.dark` variant.

### 2. Running Locally
```bash
cd portfolio-nextjs
npm install
npm run dev
# Open http://localhost:3000
```

### 3. Manual Deployment (If CI Fails)
If you need to manually deploy to VPS:
```bash
# SSH into VPS
ssh ubuntu@<VPS_IP>
cd ~/rizqibennington
git pull origin master
docker compose up -d --build
```

### 4. SSL Troubleshooting
If SSL expires or errors occur:
- Check logs: `docker compose logs certbot`
- Renew manually: `docker compose run --rm certbot renew`
- Full reset: `./init-letsencrypt.sh` (Warning: Will generate new certs).
